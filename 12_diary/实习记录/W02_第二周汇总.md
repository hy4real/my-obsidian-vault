做模型微调的尝试：
在 4 x RTX4090 上做，Qwen3.5-35B-A3B 即使是跑 4bit 的量化微调也很难做（Unsloth 不推荐 4bit），公司的 2 x A100 又在跑别的推理任务，所以目前只能尝试 Qwen/Qwen3.5-9B，LlamaFactory 不好用，还是选择用 Unsloth。

阅读了  EverMind-AI的MSA[[MemorySparseAttention.pdf]]，感觉很厉害，但是暂时不知道怎么在工业业务里用。

**在 2×4090 上微调 Qwen3.5-9B，让它能从聚合物论文中自动提取结构化 JSON 数据**。

确定用 **Qwen3.5-9B + Unsloth + bf16 LoRA** 的组合。

**冒烟测试** — 50 条数据在 2 张卡上跑通，确认环境没问题。

**数据工程** — 从 49 篇论文 + 人工审核标注的JSON 配对数据出发，先发现 23 篇切片为 0，逐步查出三种不同的 JSON 封装格式（标准/list 包裹/output 包裹）、位置索引字段类型不统一（dict/list/str）、性质类别中英文混杂，全部修复后最终产出 **3169 条训练切片**，按文献级别 8:1:1 切分。

**训练脚本** — 写了带 `train_on_responses_only`（只在 JSON 输出上算 loss）和 eval_loss 监控的正式训练脚本。

都是 claude 帮我做的，查看 claude 的 project 有完整内容。
