## 📅 Phase 1：Java后端实战（4月15日 - 5月15日）

### 你的项目：EasyShop 简化电商后台

**目标**：不是功能完整，而是**每个模块都有技术深度**，面试能讲清楚设计决策。

```
EasyShop 模块划分：
├── 用户模块  → Spring Security + JWT，角色权限
├── 商品模块  → CRUD + 分类树 + Redis热点缓存
├── 订单模块  → 状态机 + 库存扣减原子性 + 超时取消
└── 通知模块  → RabbitMQ 异步解耦，消费者幂等性
```

**技术栈**：Spring Boot 3 + MySQL 8 + Redis 7 + RabbitMQ + MyBatis-Plus

- [ ] MySQL建表（user、role、user_role）

**工作日50min分配：**
- 做题：数组 / 哈希表类型，每天1-2道 Easy→Medium
- 复习：JWT原理、BCrypt加密（10min看文章）

**本周学到的系统设计：**
> Token无状态设计为什么优于Session？RefreshToken的意义？

**里程碑：** 用户注册 + 登录接口跑通，JWT鉴权生效

### Week 2（4月21-27）— 商品模块 + Redis缓存

**周末主攻（6-8h）：**
- [ ] 实现商品模块：
  - 商品CRUD（含图片URL、SKU属性）
  - 分类管理（递归树形结构查询）
  - 分页列表（MyBatis-Plus PageHelper）
- [ ] 集成Redis缓存：
  - 商品详情缓存（Cache-Aside Pattern）
  - 缓存击穿：互斥锁（Redisson分布式锁）
  - 缓存穿透：布隆过滤器（或空值缓存）
  - 缓存雪崩：TTL随机抖动
- [ ] 数据库优化：explain分析，为常用查询加索引

**工作日50min：**
- 做题：链表 / 双指针类型（1-2题）

**本周学到的系统设计：**
> Cache-Aside vs Read-Through vs Write-Through 的权衡？  
> 为什么分布式锁解决击穿，而不用本地锁？

**里程碑：** 商品接口 + Redis缓存生效，explain无全表扫描

### Week 3（4月28日 - 5月4日）— 订单模块 + 消息队列

**周末主攻（6-8h）：**
- [ ] 实现订单模块（难点，重点讲解）：
  - 下单流程：**库存扣减 + 订单创建原子性**（MySQL事务 or Lua脚本）
  - 订单状态机：待付款 → 已付款 → 已发货 → 已完成 → 已取消
  - 超时自动取消：Redis过期key监听 + MQ延迟消息（对比方案）
- [ ] 集成RabbitMQ：
  - 交换机类型选择（Direct / Topic）
  - 生产者确认（Publisher Confirm）
  - 消费者幂等性（唯一消息ID + 数据库去重）

**工作日50min：**
- 做题：树/DFS/BFS类型（1-2题）
- 复习：事务的四个特性，隔离级别

**本周学到的系统设计：**
> 为什么用MQ做超时取消而不用定时任务扫描？  
> 消费者幂等性怎么保证？消息重复消费怎么处理？

**里程碑：** 下单流程跑通，MQ通知消费者，幂等性测试通过
