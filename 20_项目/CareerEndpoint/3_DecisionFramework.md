# 决策框架：对齐全栈AI工程师目标

当你面临一个选择时，用这个框架来评估它是否能让你离"山顶的自己"更近。

## 决策评分模型

每个选择，在以下维度打分（1-5分）：

### 维度1：技能发展（权重3倍）
这个选择能帮我学到什么技能？这个技能对"全栈AI"目标的贡献度有多大？
- [ ] AI深度（5 = 学LLM/CV等核心技能，1 = 无关）
- [ ] 系统设计（5 = 学分布式/架构，1 = 无关）
- [ ] 工程能力（5 = 学高性能系统，1 = 无关）
- [ ] 其他基础（5 = 学算法/网络等，1 = 无关）

**小计**：___/20

### 维度2：项目经验（权重3倍）
这个项目会被别人看到吗？它对秋招简历的帮助有多大？
- [ ] 可见性（5 = 生产级/开源/发表，1 = 内部）
- [ ] 复杂度（5 = 高度复杂，1 = 简单）
- [ ] 相关性（5 = 完全匹配AI/系统设计，1 = 无关）

**小计**：___/15

### 维度3：时间成本（权重1倍）
这个选择需要多少时间？占用多少其他学习时间？
- [ ] 时间投入（5 = 可控，1 = 占用太多）
- [ ] 灵活性（5 = 兼容其他计划，1 = 冲突）

**小计**：___/10

### 维度4：长期回报（权重2倍）
这个选择对2-3年后的职业有什么帮助？
- [ ] 品牌建立（5 = 能建立知名度，1 = 无帮助）
- [ ] 网络拓展（5 = 能认识牛人，1 = 无帮助）
- [ ] 职业方向（5 = 确认想走的路，1 = 无帮助）

**小计**：___/15

---

## 总分计算

```
最终分数 = (维度1 × 3 + 维度2 × 3 + 维度3 × 1 + 维度4 × 2) / 总权重(9)

得分 > 4.5：强烈推荐，这个选择对目标贡献度高
得分 3.5-4.5：可以选，有帮助
得分 < 3.5：重新考虑，可能不值得或有更好选择
```

## 常见场景决策卡

### 场景A：选择硕士课程方向

**选项1：深度学习/LLM方向**
- 技能发展：5（强化AI深度，但系统设计弱）
- 项目经验：4（有项目，但可能偏学术）
- 时间成本：4（可控）
- 长期回报：5（对AI职位帮助大）
- **总分**：约4.4（偏向选择）

**选项2：系统设计/架构方向**
- 技能发展：3（不强化AI，但强化系统）
- 项目经验：4（有项目，工程性强）
- 时间成本：4（可控）
- 长期回报：4（全栈很需要）
- **总分**：约3.8（可选择）

**建议**：混合选择——核心课程选AI，选修选系统设计

---

### 场景B：暑期实习选择

**选项1：大公司AI团队实习**
- 技能发展：5（在大牛身边学）
- 项目经验：5（生产级项目）
- 时间成本：2（很耗时，影响学业）
- 长期回报：5（简历加成最大）
- **总分**：约4.2（推荐，但需平衡学业）

**选项2：创业公司全栈实习**
- 技能发展：4（学很多，但浅）
- 项目经验：5（有多个项目）
- 时间成本：3（比较耗时）
- 长期回报：3（创业经验，但不如大厂）
- **总分**：约3.9（可选择）

**选项3：不实习，专心课程 + 个人项目**
- 技能发展：4（自主学习，有保障）
- 项目经验：3（个人项目，可见性低）
- 时间成本：5（完全可控）
- 长期回报：2（个人项目回报低）
- **总分**：约3.4（不推荐）

---

### 场景C：秋招Offer选择

**选项1：FAANG大厂**
- 技能发展：5（顶级环境）
- 项目经验：5（产品级别）
- 时间成本：4（紧张但可控）
- 长期回报：5（简历最优）
- **总分**：约4.8（强烈推荐）

**选项2：AI独角兽创业（字节、商汤等）**
- 技能发展：5（快速迭代）
- 项目经验：5（生产项目）
- 时间成本：2（压力大）
- 长期回报：4（知名度不如FAANG）
- **总分**：约4.1（推荐，但看意愿）
