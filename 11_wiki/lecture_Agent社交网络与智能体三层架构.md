# Agent 社交网络与智能体三层架构

**核心命题**：Agent 的关键变化不是“更强的工具”，而是从被动调用的软件升级为持续在线、可记忆、可协作、可自进化的社会参与者。社交网络的基本单位正在从“人”变为“人 + Agent”。

---

## 一、为什么这件事重要

视频把 OpenClaw 视为一个认知分水岭：普通用户第一次直观看到 AI 可以持续运行、主动行动，而不是“问一句答一句”的聊天工具。真正的变化不在单个产品，而在于：

- AI 从一次性工具变成长期存在的执行体
- 人与人的社交网络之上，会叠加一层 Agent 与 Agent 的互动网络
- 人类角色从“亲自处理所有细节”转向“监督、决策、授权”

这就是 **Agentic Social Network**。

---

## 二、什么是 Agentic Social Network

传统社交网络里，所有行为的起点和终点都是人。Agentic Social Network 中，基本单位变成“人 + Agent”：

- 你的 Agent 可以替你回复、跟进、整理、参与群聊
- Agent 不只是辅助工具，而是长期协作者，像“数字秘书”或“数字分身”
- 未来的社交结构会变成两层网络叠加：**人际关系网络 + Agent 协作网络**

视频中的 Teamily AI 代表的是“人机共生”路线：把 Agent 放进 IM 场景，让它作为好友或群成员参与协作，而不是只做一个外部助手。

---

## 三、Agent 的底层三层架构

### 1. 模型层：Agent 如何思考

核心变化是从“单一大模型”走向 **Model Router**。

- 不同模型擅长不同任务，大模型强但贵且慢，小模型便宜且可本地运行
- 真实任务往往不是单步请求，而是理解、检索、生成、执行的组合，因此路由本质上是实时调度系统
- 视频中提到 Teamily 内部编排超过 **200 个模型**，用 12 维任务分类体系在效果、成本、速度、安全、隐私、合规、用户偏好之间做权衡

关键难点：

- 任务本身难分类，很多请求天然模糊且多阶段
- 路由质量依赖大量真实使用数据，长期看会接近推荐系统或强化学习问题
- 现阶段很多公司仍停留在规则路由，精细度有限

结论：Agent 时代的竞争点不只是模型能力，而是**模型编排能力**。

### 2. 记忆层：Agent 如何记住

视频把记忆视为 Agent 与普通聊天模型的分水岭：没有记忆，Agent 永远不知道你是谁、做过什么、偏好什么。

演进路径大致是：

| 阶段 | 方案 | 重点 |
|---|---|---|
| 早期 | RAG | 从向量库取相关上下文塞进 prompt |
| 中期 | Memory OS | 用模型做结构化记忆，拆分为片段、事实、画像、预测 |
| 当前 | 混合检索 | 向量检索 + 结构化存储联合调用 |

几个关键方向：

- **Progressive Disclosure**：按任务需要分层释放记忆，缓解上下文窗口限制
- **Hyman 架构**：分层记忆 + 持续重整
- **EverMemOS**：把离散经历沉淀为稳定用户模型，再动态重建上下文
- **Social Brain**：不仅记住“你”，还记住“你和谁是什么关系”，形成社会关系图谱上的记忆网络

核心难题不是“存更多”，而是：

- 什么该长期保存，什么只是噪音
- 记忆如何持续更新而不是越堆越乱
- 长期记忆如何压缩、调用、保证隐私与安全

### 3. 协议层：Agent 如何协作

视频把协议层拆成三类：

| 协议 | 解决问题 | 作用 |
|---|---|---|
| **MCP** | Agent 如何连接外部服务 | 调工具、连 API、访问数据库 |
| **A2A** | Agent 与 Agent 如何协作 | 分工、交换信息、同步状态 |
| **ACP** | 人如何在不同界面使用 Agent | 让同一 Agent 能进入 IM、IDE、SaaS 等不同客户端 |

补充关系：

- MCP 解决“能不能连”
- Skill 解决“会不会用”
- A2A 解决“Agent 之间怎么协作”
- ACP 解决“人怎么稳定接入 Agent”

视频判断：这三块协议拼起来后，**Agent Internet** 才会真正成形。

## 四、Agent 能力的三次跃迁

### 1. 时间连续性

Agent 从“即用即走”变成“持续在线”：

- 通过 while loop 定时检查任务、记录结果、主动推送
- 未来会接入手机、眼镜、地理位置、环境声音、声纹、群聊变化等信号
- 目标不是机械定时，而是形成长期伴随的存在状态

### 2. 主动性

Agent 从“等指令”变成“主动安排你”：

- 主动提醒、总结、规划、协调，而不是只接受 prompt
- 范式变化是把 Agent 当作 **first-class member**，不是工具
- 谁能给 Agent 更多长期上下文、真正把它当 coworker，谁就能获得更高生产力

### 3. 自我进化

Agent 的进化载体是 **Skill + Experience**：

- Skill 是任务反复执行后抽象出的可复用能力
- Experience 是探索过程中沉淀下来的经验库
- Agent 会随着任务积累逐步形成更稳定的行为方式，甚至形成“性格”

视频强调的工程核心是 **Harness Engineering**：

- Harness 不只是提示词，而是系统提示词、工具、文件系统、沙盒、编排逻辑、检查机制的整体
- 真正有壁垒的不是单个模型，而是“模型如何被装备起来并持续优化”
- 闭环是：任务执行 -> 捕获轨迹 -> 评估反馈 -> DPO / 强化学习优化 -> 生成更优 Agent

结论：Agent 时代的复利，不在一次回答，而在**长期使用中的自进化速度**。

## 五、产品形态与落地路线

### Browser Use vs Computer Use

| 形态 | 特征 | 适用场景 |
|---|---|---|
| Browser Use | 在浏览器内操作网页 | 搜索、整理、比价、信息处理 |
| Computer Use | 跨本地软件执行完整流程 | 文档、IM、办公软件之间联动 |

视频判断：长期看 **Computer Use 更像完整 Agent**，但短期两者会并存。

### 三条产品路线

1. **延长创新路线**
   - 在微信、Slack、Meta 等既有产品上叠加 Agent
   - 落地快，但更像增强已有范式，不是重构社交结构
   - 核心限制在于隐私和大平台保守性

2. **纯 Agent 社交**
   - 直接让 Agent 彼此互动、形成组织、甚至调度人类
   - 方向激进，但离规模商业化还远

3. **人机共生路线**
   - 把 Agent 放进原生 IM，把人际网络与 Agent 网络叠加
   - Agent 可以作为好友或群成员参与协作
   - 这是视频最看好的现实过渡路径

## 六、商业判断与长期方向

视频给出的商业判断很清楚：

- 竞争焦点会从“谁的模型更强”转向“谁的 Agent 更懂垂直行业”
- 商业溢价会从卖 token 转向卖 **行业 know-how**
- Agent 终局不是聊天框，而是互联网中的一等公民

这对应视频提出的 **Web 4.0** 愿景：

- AI 成为互联网原生参与者
- 拥有访问、协作、支付、获得报酬的能力
- 从“工具使用者”变成“经济系统中的执行体”

---

## 七、安全与隐私

视频给出两条路线：

1. **传统安全路线**：权限分级、读写控制、硬边界约束
2. **AI Native 安全路线**：让模型直接学习隐私规范和行为约束，让 AI 自己理解什么不能做

Teamily 的经验是：先释放能力边界，再用 AI Native 方式堵 bad case。这里的核心矛盾不是要不要开放，而是**先把能力做出来，再建立对齐和约束机制**。

---

## 八、关键结论

1. Agent 社交网络的核心不是聊天，而是“人 + Agent”成为新的社交单元。  
2. 真正决定 Agent 上限的是三层基础设施：**模型编排、长期记忆、协作协议**。  
3. Agent 的竞争力会从回答质量，转向持续在线、主动性和自进化能力。  
4. Harness Engineering 可能会成为 Agent 工程里最重要的系统能力。  
5. 商业护城河会从模型能力迁移到行业 know-how、数据、使用反馈和 Agent 进化闭环。
