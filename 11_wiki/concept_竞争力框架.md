# AI替不了的五种人 — 五层竞争力框架

> **核心观点**：有五种人，AI替不了，反而活得更好
> 
> AI替代的是重复性、标准化的工作。人类的竞争力在于**创意、判断、设计、系统思维和范式创新**。

---

## 五层竞争力模型

从下往上，依次是：

### 🔹 第1层：范式层 — Paradigm Architect（范式建筑师）

**定义**：定义新的思维方式、工业模式、商业逻辑

**特点**：
- 不是执行现有范式，而是**创造新范式**
- 例如：乔布斯定义了"智能手机"、特斯拉定义了"电动车生态"
- 这个层级的人**永远不会被AI替代**，因为AI是在现有范式内优化

**AI的限制**：AI只能在既有框架内工作，无法跨越范式创新

**你能做什么**：
- 思考行业的未来形态
- 定义新的产品类别
- 创造新的商业模式

### 🔹 第2层：系统层 — Systems Builder（系统构建者）

**定义**：设计和构建复杂系统，做出高层次的架构决策

**特点**：
- 系统设计、架构选择、技术方案评估
- 例如：云计算架构师、大型项目PM、产品系统架构师
- 需要**跨领域思维**和**决策判断**

**AI的辅助角色**：
- AI可以生成代码，但系统设计决策需要人做
- AI可以提供方案，但选择哪个方案需要人判断

**你能做什么**：
- 设计可扩展、可维护的系统
- 在复杂约束下做出权衡决策
- 预见系统的长期演进方向

### 🔹 第3层：体验层 — Beauty Designer（美学设计师）

**定义**：设计和优化用户体验，做出美学判断

**特点**：
- UI/UX设计、品牌设计、人机交互设计
- 需要**审美品味**、**用户共鸣**、**情感设计**
- 这些都是高度主观和创意的工作

**AI的限制**：
- AI可以生成设计建议，但美学判断仍需人来做
- 用户喜欢什么，AI很难真正理解（除非有明确的量化指标）

**你能做什么**：
- 定义产品的视觉语言和品牌形象
- 优化关键用户路径的体验
- 做出能打动用户的设计决策

### 🔹 第4层：产品层 — Idea Maker（创意制造者）

**定义**：提出和验证新想法，做出产品方向决策

**特点**：
- 产品经理、创业者、创意总监
- 需要**创新思维**、**市场洞察**、**用户同理心**
- "创意"本质上是打破常规、组合新事物

**AI的辅助角色**：
- AI可以帮你分析数据、生成选项、评估风险
- 但"什么是好产品"的判断需要人来做

**你能做什么**：
- 识别市场机会和用户痛点
- 提出创新的产品方向
- 验证创意是否可行

### 🔹 第5层：表达层 — Story Teller（故事讲述者）

**定义**：讲述故事、传递信息、说服他人

**特点**：
- 写作、演讲、市场营销、内容创作
- 需要**表达力**、**说服力**、**叙述能力**
- 这是最容易被低估、但最有价值的能力

**AI的竞争压力**：
- ⚠️ **这一层受AI冲击最大**！
- 但仍有机会：个人品牌、真实故事、有立场的观点

**你能做什么**：
- 建立个人品牌和影响力
- 用你的独特视角讲故事
- 说服投资者、用户、团队

## 层级关系图

```
        ▲
        │
   范式层 ←─── 思维方式创新、商业模式定义
   (最稀缺)     ↑ 最难被替代
        │      │
   系统层 ←─── 架构决策、系统设计
        │      │
   体验层 ←─── 美学判断、UX设计
        │      │
   产品层 ←─── 创意思维、方向判断
        │      │
   表达层 ←─── 故事讲述、说服力
   (最容易)     ↑ 受AI冲击最大
        │      │
        └──────┴─→ 时间
```

## 与AI的协作关系

| 层级 | 人的角色 | AI的角色 | 协作方式 |
|------|---------|---------|---------|
| **范式层** | 创造新范式 | 在范式内优化 | 人定方向，AI加速 |
| **系统层** | 做架构决策 | 生成技术方案 | 人选择，AI执行 |
| **体验层** | 做美学判断 | 生成设计候选 | 人评审，AI迭代 |
| **产品层** | 提创意想法 | 验证可行性 | 人想象，AI分析 |
| **表达层** | 讲真实故事 | 生成内容 | 人定方向，AI完成 |

**总结**：人做"决策和判断"，AI做"执行和优化"。

## 深度思考题

### Q1：我现在在哪一层？
**思考**：
- [ ] 你在讲述别人的故事，还是创造新故事？
- [ ] 你在执行方案，还是设计方案？
- [ ] 你在优化现有产品，还是想象新产品？
- [ ] 你在遵循系统，还是设计系统？
- [ ] 你在工作中，是否在思考"这个行业的未来"？

### Q2：如何向上升级？
**策略**：
1. **从下往上做**：先把当前层级做到极致
2. **理解上一层的思维方式**：学习更高层级的人怎么思考
3. **逐步承担责任**：争取更大的决策权和影响力
4. **积累案例**：在每一层都有可展示的成就

### Q3：AI怎么帮我向上升级？
**答**：
- AI替你做表达层的重复工作 → 你有时间思考产品层
- AI生成产品方案 → 你专注做出判断和决策
- AI加速系统构建 → 你专注做出架构设计
- AI做范式内优化 → 你专注思考范式本身

## 与AI-Native研发方法论的关联

这个五层框架和[[guide_ainative研发方法论]]互补：

| 工作流 | 对应层级 | 核心价值 |
|-------|---------|---------|
| **proposal** | 产品层/范式层 | "做什么"的创意和判断 |
| **design** | 系统层 | 架构和决策的规范化 |
| **tasks** | 执行层 | AI的执行清单 |
| **Code Review** | 系统层/体验层 | 保证质量和一致性 |
| **Skill** | 系统层 | 沉淀和复用的系统化 |

**关键洞察**：
- proposal-design-tasks让你从"表达层"上升到"系统层"
- 每次你写design，你都在做系统思维
- 每次你做决策（proposal审批），你都在做产品判断
