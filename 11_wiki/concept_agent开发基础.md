## 项目选择：RAG vs Agent

### A. RAG 项目（检索增强型）

**核心特点**
- 使用向量数据库 + 检索 + 大模型的方案
- 应用：企业知识库、文档问答、垂直领域知识系统
- 难点：向量化、检索精度、提示词优化

**学习收获**
- ✅ 向量数据库工作原理
- ✅ Embedding 和相似度搜索
- ✅ LLM 与外部知识整合
- ✅ 提示词工程

**难度：** ⭐⭐⭐

---

### B. Agent 项目（自主智能体）

**核心特点**
- LLM 驱动的自主决策系统，具备行动能力
- 应用：自动化任务、复杂工作流、多步决策
- 难点：架构设计、Tool 调用管理、推理与决策

**学习收获**
- ✅ Agent 核心循环（感知 → 决策 → 行动）
- ✅ Function Calling / Tool Use
- ✅ 状态管理与上下文维护
- ✅ Agent 与后端系统集成

**难度：** ⭐⭐⭐⭐

---

### 选择建议

| 维度 | RAG | Agent |
|------|-----|-------|
| 学习成本 | 中等 | 较高 |
| 后端基础需求 | 中等 | **非常高** |
| 面试加分 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 推荐者立场 | 可选 | **强烈推荐** |

**推荐理由：** Agent 项目在大厂更有竞争力，但前提是**后端基础足够扎实**。

## 转型策略：两步走战略

### 第一步：完善 SDE（Software Development Engineer）基础

**架构与设计**
- 设计模式（23 种经典模式）
- 系统设计原理（scalability, reliability, maintainability）
- 分布式系统基础（CAP、一致性、容错）
- 数据库设计（关系型、NoSQL、缓存）

**工程能力**
- 代码质量（SOLID 原则、Code Review 文化）
- 性能优化（算法、缓存、索引）
- 测试策略（Unit、Integration、E2E）
- DevOps 与容器化（Docker、Kubernetes）

**算法与数据结构**
- 基础数据结构（Array、LinkedList、Tree、Graph）
- 常见算法（排序、查找、DP、图算法）
- 复杂度分析能力

**预估周期**
- 初级 → 中级：3-4 个月
- 中级 → 高级：6-8 个月

---

### 第二步：掌握 LLM 应用开发

**大模型基础**
- Transformer 架构（理解基本概念即可）
- Token 化与上下文窗口
- 提示词工程（Prompt Engineering）
- Fine-tuning vs In-context Learning

**Agent 框架与工具**
- LangChain / LlamaIndex 等框架
- Function Calling / Tool Use API
- Agent 设计模式（ReAct、Chain of Thought 等）
- 向量数据库集成

**工程实践**
- 错误处理与重试机制
- Token 成本优化
- 并发调用管理
- 日志与监控

**预估周期**
- 入门：2-3 周
- 精通：2-3 个月（基于 SDE 基础之上）

---

### 完整转型时间线

```
Month 1-4: 完善 SDE 基础（系统设计、算法、工程原理）
     ↓
Month 5-6: 学习 LLM 基础与 Agent 框架
     ↓
Month 6-8: 完成 1-2 个完整的 Agent 项目
     ↓
Month 8-9: 面试准备与技术文章撰写
     ↓
Month 9+: 申请高级工程师职位（Sept internship）
```

## 大厂面试重点

### 三轮面试考察体系

#### 第一轮：基础轮（筛选层）

**考察内容**
- **LeetCode 难题** - Medium ~ Hard（20-30 分钟内完成）
- **系统设计基础** - 单机架构设计
- **编程实践** - 代码风格、错误处理、SOLID 原则
- **问题排查** - 系统故障定位、性能瓶颈分析

**准备建议**
- ✅ 完成 100+ LeetCode Medium/Hard
- ✅ 练习系统设计 5-10 个场景
- ✅ 学习主流架构（Microservices、Message Queue）
- ✅ 阅读开源项目源码

**打分标准**
- ⭐⭐⭐⭐⭐ 满分：30 分钟内完成、代码质量高、思考深入
- ⭐⭐⭐ 通过：40+ 分钟完成、有优化思路、逻辑清晰
- ⭐ 不通过：无法完成或明显逻辑错误

---

#### 第二轮：项目轮（项目经验评估）

**考察内容**
- **AI/LLM 项目经验** - 架构设计、技术选型理由、问题解决案例
- **Agent 设计理解** - 核心循环、Function Calling 设计、状态管理
- **性能优化** - Token 成本优化、并发调用管理、错误恢复
- **工程质量** - 代码组织、测试覆盖、部署与监控

**准备建议**
- ✅ 完成 1-2 个完整的 Agent 项目
- ✅ 准备 15-20 分钟的项目演讲
- ✅ 能够深入讲解技术选择与权衡
- ✅ 有开源贡献或 GitHub portfolio

---

#### 第三轮：重点轮（高级能力评估）

**考察内容**
- **多轮决策系统设计** - 复杂推理、上下文维护、反思与自纠正
- **数据库与存储** - Schema 设计、查询优化、事务管理、向量 DB
- **并发与分布式** - 高并发 Agent 调用管理、分布式系统设计
- **系统架构** - 端到端设计、监控日志告警、成本与性能权衡
- **业务理解** - 为什么这样设计？替代方案？如何改进？

**准备建议**
- ✅ 深入学习一个数据库（Postgres 或 MongoDB）
- ✅ 学习 Redis/Memcached 缓存策略
- ✅ 理解 Kafka/RabbitMQ 消息队列
- ✅ 准备 2-3 个深度项目讲解

## 决策树：如何选择转向

### 三维度评估

**①代码能力**
- 优秀：LeetCode Hard 题 20 分钟内解决
- 一般：LeetCode Medium 题 30+ 分钟

**②数学基础**
- 扎实：能理解复杂算法背后的数学原理
- 薄弱：主要凭直觉和经验编程

**③目标公司**
- 大厂：高竞争、全能要求、系统的 3-4 个月准备
- 创业：快速交付重于理论深度、重视开源和实战

### 四种典型场景

#### 场景 1：代码优秀 + 数学扎实 + 目标大厂
**评价：** ⭐⭐⭐⭐⭐ **最优选择**  
**成功概率：** 80%+ 拿到 offer  
**推荐时间线：** 4 个月冲刺

#### 场景 2：代码一般 + 数学薄弱 + 目标大厂
**评价：** ⚠️ **需要充分准备**  
**成功概率：** 40-50%  
**建议：** 先从创业公司或中型公司开始

#### 场景 3：代码优秀 + 任何基础 + 目标创业
**评价：** ⭐⭐⭐⭐ **快速上手**  
**成功概率：** 90%+  
**推荐时间线：** 2-4 周快速原型

## 避坑指南：三大致命陷阱

### ⚠️ 陷阱一：底层框架选择陷阱

**问题：** 花 2-3 个月纠结框架选择，不写代码

**正确做法**
- ✅ 在 1 周内快速选择一个流行框架
- ✅ 立即开始项目，在实战中学习
- ✅ 遇到框架限制时再考虑替换

**框架建议**
- 小项目 + 快速原型 → **LangChain**
- 大型系统 + 高可靠性 → **LlamaIndex / CrewAI**
- 学习用途 → **任选，重点是理解概念**

---

### ⚠️ 陷阱二：知识体系缺失陷阱

**症状**
- ❌ 能跑通 Demo，但遇到实际问题就卡壳
- ❌ 面试时讲不清楚设计原理
- ❌ 系统性能差，不知道怎么优化

**必须掌握的知识体系**
- 数据库基础 + 缓存策略
- Agent 核心循环 + Function Calling
- 上下文窗口管理 + Token 计数
- 错误恢复 + 性能优化

**短期补救**
1. 梳理当前项目的设计文档
2. 写出可能的问题列表（Token 溢出？并发问题？）
3. 针对每个问题阅读相关教材或源码

---

### ⚠️ 陷阱三：学习路径错误陷阱

**错误路径 ❌**
```
Transformer 论文 → LLM 原理 → 提示词工程 → Agent 框架
耗时：2-3 个月 | 收益：理论知识，实战技能 0
```

**正确路径 ✅**
```
系统设计基础 → Agent 框架 → 完整项目 → 性能优化
耗时：4-6 个月 | 收益：实战能力 + 深度理解
```

**学习速度对标**
- **优秀** - 4 个月完成全套（Month 1: 基础, Month 2: 框架, Month 3: 项目, Month 4: 面试）
- **正常** - 5-6 个月完成
- **缓慢** - 8-9 个月（在框架选择、理论学习中徘徊）

### 编程语言

| 语言 | 地位 | 推荐指数 | 适合人群 |
|------|------|---------|---------|
| **Python** | AI/ML 标准语言 | ⭐⭐⭐⭐⭐ | AI 工程师首选 |
| **Go** | 高并发系统新星 | ⭐⭐⭐⭐ | 中大型项目 |

### 应用框架

**LangChain** - 最成熟、最广泛（80k+ GitHub Star）
- 优势：社区活跃、文档完整、生态丰富
- 劣势：学习曲线陡峭、版本更新快

**LlamaIndex** - 专注 RAG 和数据索引
- 优势：RAG 支持最完整、性能优化好
- 劣势：社区规模小于 LangChain

**CrewAI** - 多 Agent 协作框架（新兴）
- 优势：协作机制清晰、代码简洁
- 劣势：社区较新、生产稳定性未充分验证

## 核心洞察

> **后端基础是前提** — Agent 开发的难度 80% 来自系统设计，20% 来自 LLM

> **学习路径要正确** — 不是"理论 → 框架 → 项目"，而是"基础 → 框架 → 深度项目"

> **项目实战是唯一验证** — 1 个完整的项目 > 3 个 Demo，项目中的问题会暴露所有知识缺陷

## Agent 系统的 5 层地基

> 来源：十年老技术开发的 AI Agent 探索之路（2026-05-07）

| 层 | 内容 | 缺失后果 |
|---|------|---------|
| 1 | 目标表达——到底想完成什么 | 系统无方向 |
| 2 | 能力单元——有哪些 Skill/工具/工作流 | 无法执行 |
| 3 | 运行时状态——当前正在做什么 | 无法观测 |
| 4 | 治理边界——允许做什么，不允许做什么 | 无法控制 |
| 5 | 评估反馈——哪些行为值得固化，哪些必须修正 | 无法进化 |

> 少任何一层，系统都可能看起来能跑，但跑不稳。

## 脚手架 > 模型

这是最关键的一个工程判断：

| 投入 | 成本变化 | 效果变化 |
|------|---------|---------|
| 模型升级 | +300% | +20% |
| 脚手架升级 | +50% | +200% |

> **垃圾的思考乘以强大的模型，等于精美的垃圾。** 一个设计精良的系统让弱模型发挥惊人性能，烂系统完全浪费顶级模型。

优先投资：Observability > Eval > Skill 沉淀 > 模型升级。

## 24h 打工人架构：文件 + 轮询

自建调度层的核心设计（适合 Agent 高速迭代期）：

- **存储**：文件系统（出问题直接让 AI 看文件，方便 Git 版本控制）
- **调度**：轮询队列 + 失败自动切换工具（codex → gemini → claude）
- **并发**：组间并行（前端/后端并行）+ 组内串行（避免文件冲突）

> 文件系统 + JSON 状态比数据库更适合早期 Agent 系统。等系统稳定到需要事务和并发锁时再升级。
