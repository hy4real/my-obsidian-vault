# 罗福莉访谈：AI 范式已然巨变

**核心命题**：2026 年大模型战争从 Pre-train 主导的 Chat 时代，转入 Post-train 主导的 Agent 时代。Anthropic 路径成为共识，国内厂商加速追赶。

---

## 一、OpenClaw 引发的范式巨变

**OpenClaw** 是一套开源、可二次修改的 Agent 框架，关键差异在于：
- Claude Code 是黑盒（专为软件工程设计）
- OpenClaw 开源 → 可以让模型**重写自己的 memory 系统、改 multi-agent 逻辑**
- 这种可改性催生**群体智能**：大量用户同时迭代，几小时一个循环

**框架的本质**：不是 UI 或产品，是感知模型长短板、做成本调度、补足行动 context 的**协调层**。一个好框架的判断标准是：弥补行动短板（memory、message channel、主动性、自更新）和具备可泛化的评估体系。

**Skill 的意义**：Pre-train 数据里没有的企业内部规范，通过 Skill 形式补充给模型，提升成功率。

## 二、2026 竞争格局

| 玩家 | 路线 |
|---|---|
| 豆包 / 元宝 / 阿里 | 打 DAU，互联网产品打法 |
| Kimi | 走 Anthropic 路线（Agent 能力优先）|
| 罗福莉 | 不在乎 DAU，只在乎 Agent 能不能替代自己 |

**AGI 时间预期**：从"两年以上"提前到"两年以内"。触发因素：**AI 训 AI** 已成为可能（Skill 化 + Workflow 化覆盖了原以为有创造力的工作）。

### 护城河演化（跨视角补充）

OpenClaw / Claude Code 等 Agent 工具的崛起，同步引发了一个连锁判断——来自 [[lecture_空间智能与AI的下一步]] 黄晓煌的认知迭代：

> 早期：工具 + 数据 + 算法模型形成飞轮
> Agent 工具出现后：**纯工具不再是壁垒**（Agent 可以直接生成/替代下游应用层）
> 现在：**模型 + 硬件连接生态 + 数据** 才是核心护城河

对 SaaS / 纯工具公司是巨大警示。罗福莉在本访谈中印证了同一逻辑：她说框架的价值在于**弥补模型短板、做成本调度**，而非提供功能——功能本身会被快速复制。

## 三、MiMo-V2 架构要点

小米发布四模型：**Flash / Pro / OMNI / TTS**，在 Agent 复杂架构下速度超预期。

### Pro（约 1T 参数）关键决策

| 决策 | 说明 |
|---|---|
| **Hybrid Attention** | Full attention : Sliding-window = 1:7；大模型可承载更高稀疏度；**层数比比例更重要** |
| **MTP（多 Token 预测）** | 预训练用 1 层提升基座；推理阶段多层加速；命中后 verify，不带来幻觉 |
| **不选 MLA** | MLA 在 Chat 时代优秀，但计算已压到极限，上 MTP 会被计算 bound |
| **TPS 60–150** | 用户能感知到速度提升 |

### 多模态处理
- **Audio**：音频离散化为 token（多层 RVQ），用 LLM 范式建模，泛化出强风格化能力
- **TTS**：统一离散架构，少量 SFT/RL 即可遵循复杂风格描述
- **OMNI 比 Pro 小，但情商和世界知识更强**

### 定价逻辑转变
- Chat 时代：按推理成本定价
- Agent 时代：按**模型产生的最终价值**定价

---

## 四、训练 1T 的挑战

- **1T 参数 + 高效非注意力架构 + 强 Code 预训练** = Agent 时代入场券
- 真正挑战不是数据（大模型对脏数据容忍度高），而是**发现并解决问题**：
  - 把 loss spike 当严重问题处理（而非视为正常）
  - 监控每层激活值、expert 负载、QK logits 异常
  - 工具箱：LayerNorm 压制、Gradient Clip、QK Clip

**算力分配对比**：
- Chat 时代：研究 : Pre-train : Post-train ≈ 3:5:1
- Agent 时代：≈ 3:1:1（研究投入 > 训练总和）

## 五、组织与人才观

- **100 人，无小组、无职级**，Pre-train 人才自然流向 Post-train
- **招人偏向大二大三**：对 Agent 范式无固有偏见，想象力强，1-2 个月习得能力
- 核心判断：**"环境比经验更重要"**

---

## 六、几个反直觉观点

1. **MultiAgent 目前有点伪**：能省成本提速，但还没看到真正拉高任务完成率上限
2. **Coding 是三个范式的共同红利**：长依赖密集 → 长 context 建模；有 verify 信号 → RL 友好
3. **AI 没有生存危机**：模型进化路径不会复刻人类，会更自由、更有创造力

## 专有名词注释

| 术语 | 解释 |
|---|---|
| **Pre-train（预训练）** | 用海量通用数据训练模型基础能力的阶段，决定模型"上限" |
| **Post-train（后训练）** | 预训练之后，用 SFT/RL 等方式针对特定目标优化，决定模型"落地效果" |
| **Agent** | 能自主规划、调用工具、多步执行任务的 AI 系统，区别于只问答的 Chat 模型 |
| **Skill** | Agent 框架中可复用的能力模块，相当于"工作流函数"，补充 Pre-train 数据中没有的企业规范 |
| **Context（上下文）** | 模型单次处理的文本范围，越长越贵但记忆越多 |
| **DAU** | Daily Active Users，日活跃用户数，互联网产品核心增长指标 |
| **MiMo-V2** | 小米发布的大模型系列，含 Flash/Pro/OMNI/TTS 四个版本 |
| **Hybrid Attention（混合注意力）** | 混合使用全注意力（精准但慢）和滑动窗口注意力（快但局部）的架构 |
| **Full Attention** | 每个 token 关注所有其他 token，计算量 O(n²)，精准但昂贵 |
| **Sliding-window Attention** | 每个 token 只关注附近固定窗口，计算量线性，速度快 |
| **MTP（多 Token 预测）** | 一次预测多个后续 token，加速推理，命中后需 verify |
| **MLA（Multi-head Latent Attention）** | DeepSeek 提出的注意力变体，压缩 KV Cache 节省显存，但计算已到瓶颈 |
| **TPS（Tokens Per Second）** | 每秒生成的 token 数，衡量推理速度 |
| **RVQ（残差向量量化）** | 将连续音频信号离散化为 token 序列的技术 |
| **Loss Spike** | 训练中损失函数突然异常飙升，通常意味着训练不稳定 |
| **MoE（混合专家）** | 模型每次只激活部分"专家"子网络，大幅提升参数效率 |
| **RL（强化学习）** | 通过奖惩信号训练模型，o1/R1 的核心技术 |
| **Rollout** | 强化学习中模型执行一段完整推理/行动序列的过程 |
| **Checkpoint** | 训练过程中保存的模型状态快照，可用于恢复或评估 |
| **SWE-bench / AIME** | 软件工程任务 / 数学竞赛题的标准评测集，用于衡量模型能力 |
| **Agent RL Scaling** | 在 Agent 任务上应用强化学习规模扩展，当前最热竞争点 |
| **LayerNorm / QK Clip** | 训练稳定性技巧，防止梯度爆炸或数值溢出 |
| **OpenClaw** | 开源可修改的 Agent 框架，类 Claude Code 但允许用户重写核心逻辑 |
