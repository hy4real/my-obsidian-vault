# 【闪客】大模型已死？上帝视角拆解三年 LLM 架构演进！

> [!info] 视频信息
> 飞天闪客 / 11:23 / 2026-05-09 / [原视频](https://www.bilibili.com/video/BV1mxR9BiEm8/)

## 核心观点

**单靠继续优化 Transformer 架构来突破模型能力上限的空间已经很小了。** 作者梳理了 2019 年 GPT-2 到 2026 年 DeepSeek-V4 共 67 个核心开源模型的架构变化，结论是 Transformer 五个核心模块（位置编码、归一化、残差连接、FFN、注意力）能改的花样基本都被挖干净了。开源/闭源差距持续缩小，本身就是发展放缓的信号。

## 视频结构

> [!note] 上帝视角的拆解路径
> 把 Transformer 拆成 5 个模块，逐个"宣判死刑"：
> 1. 位置编码（Positional Encoding）
> 2. 归一化（Normalization）
> 3. 残差连接（Residual Connection）
> 4. 前馈神经网络（FFN）
> 5. 注意力（Attention）

## 模块 1：位置编码 — 已死

| 阶段 | 方法 | 来源 |
|------|------|------|
| 起点 | 固定位置编码（正余弦） | Transformer 原论文 |
| 主流 | 旋转位置编码（RoPE） | 让点积具备友好的位置特性 |
| 扩展 | YaRN | 支持更长上下文 |
| 极端 | NoPE（无位置编码） | 在某些场景被证明有效 |

**结论**：技术路线基本收束，能玩的都被玩过了。

## 模块 2：归一化 — 已死

> [!abstract] 归一化的设计自由度
> - **算法**：LayerNorm → RMSNorm（更适合大模型）
> - **位置**：Pre-Norm / Post-Norm / 残差前 / 残差后 / 前后都加
> - **作用对象**：词向量、Q（QNorm）、K（KNorm）、V（VNorm）、QK（QKNorm）、KV（KVNorm）

所有可能的组合都被组合过了。

## 模块 3：残差连接 — 刚被动刀

| 角色 | 创新 |
|------|------|
| 原始 Transformer | 简单加法，多年无人敢动 |
| 字节 / DeepSeek / Kimi | 提出 HC、mHC、AttnResidual |

**状态**：刚被切开探索，花样有限，仍处于早期阶段。

## 模块 4：FFN — MoE 化已成标配

```
原始：一个大 FFN
   ↓ 拆分
MoE：N 个小 FFN（专家） + 路由层
   ↓ 再优化
DeepSeek-MoE：细粒度拆分 + 共享专家
```

> [!tip] 命名规则
> 模型名里的 `A37B-A17B` 这种写法 = 总参数 37B，激活 17B。
> Dense（稠密）→ Sparse（稀疏 / MoE）已是大势所趋。

## 模块 5：注意力 — 花样最多，本质却简单

> [!summary] 注意力机制的三条优化路径
> 注意力的核心痛点：每个 token 都要和其他 token 算一次。三条解法对应三类创新：

| 思路 | 含义 | 代表方法 |
|------|------|----------|
| 缩短向量 | 数量不变，单次更便宜 | MQA、GQA、MLA |
| 减少计算对象 | 只算窗口内 / 挑选部分 token | DSA、SWA、CSA、HCA（稀疏注意力） |
| 改成线性复杂度 | 复杂度由 O(n²) 降到 O(n) | KDA、Gated DeltaNet、Lightning、Mamba |

**Hybrid Attention**：传统注意力 + 线性注意力混用，目前百花齐放，无统一胜者。

## AI 发展的三个定律

> [!warning] 三大定律一致指向"放缓"
> 这是作者最关键的宏观判断。

1. **Scaling Law（大力出奇迹）** — 互联网数据被挖干净，算力和模型规模短期难再翻倍，遇到瓶颈。
2. **Bitter Lesson** — 人类在架构层加的小巧思，长期看会成为更大模型的阻碍。"人类的自以为是"会拖慢模型学习速度。
3. **Moravec 悖论** — 人类擅长的对机器难，机器擅长的对人类难。Transformer 这个底层架构本身可能就不是正确答案。

## 关键判断

- **差距收缩 = 增长放缓**：
  - 线性发展 → 差距不变
  - 指数发展 → 差距越来越大
  - 差距收缩 → 一定是发展放缓

  开源 vs 闭源、各家厂商之间的差距都在缩小，与"放缓"假设一致。
- **未来方向**：多模态、世界模型，但当前多模态多数仍在做"模态对齐 + 套用 LLM"，世界模型边界尚未清晰。

## 原始转录

> [!note]- 完整转录（约 5800 字）
> 你觉得大语言模型发展到尽头了吗？为了搞明白这个问题，我梳理了从 2019 年的 GPT-2 一直到 2026 年的 DeepSeek V4 共 67 个核心开源模型的架构变化。我发现这个 Transformer 架构的底座似乎已经没有多少新花样可以玩了，同时模型与模型间的差距也在逐渐缩小。比如说你有没有感觉似乎好久没有遇到一个让你感觉像当时 GPT-4 那样震惊的模型了？那这是否意味着语言模型走到头了呢？今后的架构创新还有新的机会突破吗？更进一步讲，AI 的发展是否已经到瓶颈了呢？今天这期视频我就从上帝视角带你看懂近三年来核心架构的演进，放心没有任何复杂的公式，我会用最清晰、最直白、最不啰嗦、最不废话、最不绕弯子的语言给你彻底讲明白，我们开始吧。
>
> 自 ChatGPT 出现后到现在已经三年多了，所有我们熟知的大模型都只是 Transformer 架构的一个分支而已，所以 Transformer 仍然是个绕不开的话题。如果你还不熟悉它没有关系，我给你个最简单但却很准确的简化版本：输入和输出都是文字，也就是 token，开头有个编码器，就是把每个文字都转换成计算中的向量，末尾有个解码器，就是反过来把计算机中的向量转换成文字，完成预测下一个词这样的任务，中间就是对这些向量进行各种计算，其中注意力层就是向量间的加权求和，目的是让每个词都包含其他上下文的信息，前馈神经网络层就是给每个 token 的向量做个非线性变换，这一步就是最终能计算出下个词是什么的关键步骤，就这么简单。你说这破玩意儿能撑这么多年？没错，那接下来我们就一起来看看这些年人们都在这个破玩意上缝了多少针吧。
>
> 先看编码器这部分。在原始的 Transformer 架构中分成 token embedding 和 positional embedding 两部分，这里的位置编码就是在原向量的基础上，根据不同的词的位置再加上一个新的向量。那具体加的这个向量是什么？Transformer 原论文中是取个固定的值，具体说是通过正余弦函数算出一组值，这种方式称为固定位置编码，但是后来被一种叫做旋转位置编码的技术给淘汰掉了，这种方式可以让向量间计算点积时得到一些友好的位置特性。再后来又有人发明了对旋转位置编码的扩展方法，目的是可以支持更长的上下文位置计算，甚至还有人提出了一种叫 NoPE 即没有位置编码的一种位置编码方式，而且还证明了一些场景的有效性。看到了吧？就这么一个位置编码的创新基本就被挖干净了，实际的应用也基本是事件线收束了。好了，我宣布——位置编码死。
>
> 回到这里再看个更夸张的。有的时候我们为了稳定训练，需要把向量控制在一个可控的范围内，同时要保证它们之间相对的差距不变，比如说原来是这个样子，那把均值控制为零，方差控制为一就变成了这个样子，这种把数据变换到一个更稳定的范围的方法叫做归一化。在 Transformer 原始论文中具体的归一化方法叫 LayerNorm 即层归一化，而现在大模型更常用的一种叫法叫 RMSNorm，更适合当代大模型的训练。还没完，这只是具体计算算法的不同，而这个归一化层还可以安插在模型中的各种位置，比如说安插在注意力层之前叫做 pre-norm，安插在注意力层之后叫做 post-norm，那还可以前面也来一个后面也来一个。还没完，接下来我们还会讲到残差连接，这个 post-norm 还可以像这样安插在残差连接之前，也可以像这样安插在残差连接之后，这样组合起来可就老多了，但是不好意思，所有这些组合都被咱们伟大的人类玩过了。还没完，这个归一化操作可不仅仅是作用于词向量，还可以作用在注意力层里面的这个 QKV 向量上，那用在 Q 向量上就叫做 QNorm，用在 K 向量上叫做 KNorm，用在 V 上又叫 VNorm，然后还可以同时作用在 QK 上叫做 QKNorm，同时作用在 KV 上叫做 KVNorm，不好意思，这些组合也全都被玩完了。那你还能想到其他改动的点吗？归一化死。
>
> 来看看下个倒霉鬼是谁吧。这俩大哥咱们先别动，刚刚我们不是提到了个残差连接嘛，在原始的 Transformer 实现中就是一个简单的加法操作，这么多年来也没人敢动这里。但是后来咱们的字节、DeepSeek、Kimi 分别提出了 HC、mHC、Attention Residual，让这个所有人都觉得是个默认的、肯定是正确的部分也被动了刀子。当然了这块还比较新，目前还处于探索阶段，玩的花样也十分有限。残差连接死。
>
> 接下来就剩这俩大哥了，让我们先给这个占地面积最大的 FFN 层动刀吧。这个 FFN 层其实就是一个普通的全连接神经网络，属于是深度学习入门级别的结构了，只不过在大模型这里面就占地面积特别大了，这里的每条线都可以表示一个参数，那如果它特别大的话就很占。怎么办呢？答案很简单——变小点就好。那我们这样，先把这个大的 FFN 拆成两个小的 FFN，最后增加一个可训练的路由层，每个 token 的计算只路由到其中一个 FFN 上，这样就能保证总参数量仍然是很多，但是推理时只激活少量的参数，做到了一种平衡。这里的每个小 FFN 的网络又叫做 expert 专家，整个结构构成的就叫做混合专家模型 MoE。后来 DeepSeek 又对这个网络进行了再次优化，比如说把专家拆分得更细，引入了共享专家等等，起了个新的名字，很粗暴，就叫做 DeepSeek-MoE。
>
> 现在你在很多模型名字上看到类似什么 A 多少多少 B 这种写法就是 MoE 架构，表示总参数量是 397B，每次只激活 10B、17B 的意思。现在的大模型基本上都 MoE 化了，只是专家数和激活比例各有各的配法。为此人们还发明了个新词，把之前所有参数都参与计算的模型叫做 dense 稠密模型，现在只激活部分参数的叫 sparse 稀疏模型，其实就是 MoE。死。
>
> 好了，就剩下这个最经典的注意力层了，是 Transformer 架构的一个精髓，而且人们在这上花的功夫可以说是恐怖如斯，我就没见过这么卷的。有 Transformer 原始论文中的 MHA 以及后来改良的 MQA、GQA，还有 DeepSeek 研发的 MLA，然后还有稀疏类型的，比如说 DeepSeek 的 DSA、SWA、CSA、HCA 等等，然后以及把注意力改成线性的，比如说 Kimi 的 KDA、Gated DeltaNet、Lightning Attention，还有现在最新的 Mamba 模型，这些全都是注意力机制的改良。虽然看着眼花缭乱，但本质上解决的问题却非常简单，我一说你就懂了。
>
> 其实这个注意力层最大的问题就是每个 token 向量都需要和其他向量来一次计算。一种办法是仍然每个 token 之间都算一次，但是计算量相对减少一点，比如说想办法把向量缩短等等，这就是 MQA、GQA、MLA 所努力的方向，数量没变，但是难度降低了。另一种办法是计算的数量减少，比如说只计算一个固定窗口内的 token，或者通过某种办法挑选一部分 token 进行计算，这就是 DSA、SWA、CSA、HCA 思想的核心，也叫做稀疏注意力类型。还有另一种更极端的办法就是直接把注意力机制的复杂度变成线性增长，比如说 KDA、DeltaNet、Lightning、Mamba 都属于这种。由于它实在太另类了，所以人们把传统注意力和这种线性注意力混合使用的方式又叫做混合注意力 Hybrid Attention，这块暂时就没有那么统一了，现在是百花齐放，谁也不知道哪个更好。但是也是由于实在太卷了，能创新的点早就被挖干净了，Attention 死。
>
> 总结一下：位置编码、归一化、残差连接、FFN、注意力，整个架构已经没有多少可以动刀子的地方了。当然这期视频有点标题党，严谨点说是：我觉得现在的语言模型领域，单纯依靠继续优化 Transformer 架构而带来模型能力上限的突破，这条路的空间已经不大了。从一年一度的斯坦福报告中也可以看出，开源模型和闭源模型的差距在不断缩小，各厂商之间的差距也在不断缩小。这其实很好理解，假如大模型的发展是线性的，那它们的差距应该是保持不变的；假如是指数型的，它们的差距反而应该是越来越大；只有发展是逐渐放缓的，差距才会逐渐缩小。同时这也和我们的体感是差不多的，能大幅提升性能的简单手段真的是已经越来越少了。
>
> 那从更宏观的视角，我找到了 AI 发展的三个定律。Scaling Law 就是我们说的大力出奇迹，但是现在互联网的数据基本上已经被咱们人类挖干净了，算力和模型规模也很难在短时间内提升，所以遇到了瓶颈。Bitter Lesson 告诉我们：从长期来看，我们人类人为在模型架构层面设计的一些小巧思，在更大的模型面前反而是阻碍，更狠一点说就是人类的自以为是阻碍了模型的学习速度。尤其是在现在大模型架构已经被人们动了这么多刀了，即便是有新的想法，在这个定律面前也可能是一种负优化。莫拉维克悖论就是说对人类很简单的，对机器很难，反过来对机器很简单的，对人类反而很难，比如说机器在算术方面就秒杀人类，但是你让它缝个衣服试试，这就是世纪难题了。即便是在语言模型中这个问题依然存在，整个底层的这个 Transformer 架构其实并不一定是正确的。
>
> 当然了基于这些考虑，人们也早就在尝试其他的做法了，比如说多模态和世界模型，只不过现在有一些多模态它只是做了一个模态对齐，最终还是在利用大语言模型的便利，世界模型那就更模糊了，边界都不清晰，这个我可以找时间单独出一期视频好好聊聊，这里混乱得很。
>
> 不知道你看到这里有什么感受呢？这里我也在视频最后稍稍叠个甲：虽然视频中我之前有各种什么这个死了那个死了，其实并不是对技术的不尊重，我恰恰是想用这种方式给大家形象地展示一下现在大模型的一个架构变迁。其实我们每天接触的新名词可能有很多，加上每一次一个新的技术出现的时候总会带来一些过度的宣传。但是本期视频你就会发现，归根结底其实也就是一堆固定的套路反过来调过去地改，很少有完全出乎意料的一些花样。我觉得我们不应该高估单个技术的贡献，同时也不应该低估技术的量变产生质变的一个过程。等后面再出现什么新技术的时候，你可以回看一下这期视频，从上帝视角重新审视一下，或许你能很快判断出这些新技术的地位和影响。
